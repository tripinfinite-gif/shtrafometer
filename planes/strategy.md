# Стратегия разработки SiteChecker

> Комплексный план реализации сервиса анализа сайтов на нарушения законодательства РФ.
> Основан на: исследовании рынка + rf_website_compliance.md (55 пунктов чек-листа, 28 категорий штрафов).
> Каждая фаза спроектирована для параллельного выполнения независимыми агентами.

### Стек технологий

| Слой | Технология | Почему |
|------|-----------|--------|
| Фреймворк | Next.js 16 (App Router) | Фронт + бэкенд в одном проекте |
| Язык | TypeScript | Типизация, меньше багов |
| UI | React + Tailwind CSS 4 | Быстрая вёрстка |
| API | Next.js Route Handlers | API внутри проекта |
| Парсинг HTML | cheerio | Быстрый DOM-парсинг без браузера |
| HTTP | fetch (Node.js 24 native) | Скачивание страниц |
| Runtime | Node.js 24 | Уже установлен |

## ТИПЫ ДАННЫХ (types.ts)

```typescript
interface CheckRequest {
  url: string;
}

interface CheckResponse {
  url: string;
  checkedAt: string;
  totalMinFine: number;
  totalMaxFine: number;
  riskLevel: 'low' | 'medium' | 'high' | 'critical';
  siteType: 'informational' | 'ecommerce' | 'service' | 'unknown';
  violations: Violation[];
  warnings: Warning[];        // Рекомендации (не автоматизируемые проверки)
  passed: PassedCheck[];
  stats: {
    totalChecks: number;
    violations: number;
    warnings: number;
    passed: number;
  };
}

interface Violation {
  id: string;                 // "pd-05"
  module: string;             // "personal-data" | "localization" | ...
  law: string;                // "152-ФЗ"
  article: string;            // "ст. 13.11 ч.2 КоАП"
  severity: 'critical' | 'high' | 'medium' | 'low';
  title: string;
  description: string;
  minFine: number;
  maxFine: number;
  details: string[];          // Конкретные находки
  recommendation: string;
  reference: string;          // Ссылка на текст закона
}

interface Warning {
  id: string;
  title: string;
  description: string;
  law: string;
  potentialFine: string;      // "до 10 000 000 ₽"
  recommendation: string;
}

interface PassedCheck {
  id: string;
  title: string;
  module: string;
}
```

**Статус: НЕ ВЫПОЛНЕНА**

Создать файлы модулей (каждый модуль — чистая функция, получает cheerio DOM и raw HTML, возвращает массив Violation[]):

- `src/checks/mod-personal-data.ts` — проверки pd-01, pd-02, pd-05, pd-09, pd-10, hidden-01, hidden-02
- `src/checks/mod-localization.ts` — проверки loc-01 через loc-11 (зарубежные сервисы — самые дорогие штрафы!)
- `src/checks/mod-language.ts` — проверка lang-01 (иностранные слова в интерфейсе)
- `src/checks/mod-consumer.ts` — проверки con-01, con-02, con-03, con-05, con-07
- `src/checks/mod-security.ts` — проверка sec-01 (HTTPS/SSL)
- `src/checks/engine.ts` — оркестратор: принимает URL, делает fetch, создаёт cheerio DOM, запускает все модули параллельно, собирает результат, считает итоговые штрафы, определяет riskLevel

Логика engine.ts:
```
1. Нормализовать URL (добавить https:// если нет)
2. fetch HTML
3. Определить тип сайта (ecommerce / informational / service)
4. Создать cheerio DOM
5. Promise.all([
     checkPersonalData(dom, html),
     checkLocalization(dom, html, url),
     checkLanguage(dom),
     checkConsumer(dom, html, siteType),
     checkSecurity(url),
   ])
6. Объединить violations + passed + warnings
7. Рассчитать totalMinFine, totalMaxFine
8. Определить riskLevel
9. Вернуть CheckResponse
```

Определение типа сайта:
```
ecommerce: если найдены: «корзин», «купить», «в корзину», «оформить заказ», «каталог», «товар», cart, checkout, «добавить в», цена + ₽/руб
service: если найдены формы заказа услуг, но нет каталога товаров
informational: всё остальное
```

## КОНКУРЕНТНОЕ ПОЗИЦИОНИРОВАНИЕ

### Что мы делаем лучше всех

1. **Комплексность**: 8 законов, 35+ проверок, 28 категорий штрафов — ни один конкурент не покрывает всё
2. **Калькулятор боли**: итоговая сумма штрафов с разбивкой по законам
3. **Бесплатно**: MVP полностью бесплатный (конкуренты берут 300-5000₽)
4. **Скорость**: результат за 5-10 секунд (не 2 минуты как у rffz.ru)
5. **Неочевидные нарушения**: проверяем то, что другие пропускают (остатки GA, Google Fonts через CDN, пассивное согласие, предзаполненные чекбоксы)
6. **Штрафы за локализацию**: 1-6 млн₽ за каждый зарубежный сервис — это самые дорогие штрафы, и мы их показываем первыми

### Ограничения MVP

| Ограничение | Влияние | Когда решим |
|------------|---------|-------------|
| Только fetch (нет headless browser) | SPA-сайты покажут пустой HTML | Фаза 5+ (Playwright) |
| Только главная страница | Политика может быть на отдельной странице | Фаза 5A (краулинг) |
| Эвристики (не AI) | Возможны false positives | Фаза 4 (калибровка) |
| Нет проверки реестра РКН | Не проверяем уведомление оператора | API не документировано |
| Нет анализа текста политики на полноту через AI | Только поиск ключевых слов | Будущее (LLM) |
