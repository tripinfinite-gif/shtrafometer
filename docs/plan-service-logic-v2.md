## 1. Концепция: "Покажи проблему → предложи решение"

Главный принцип: **пользователь должен увидеть ценность ДО регистрации**, а регистрация — это мост к решению его проблемы, а не барьер.

### Текущая модель (v1)
Все результаты показываются бесплатно → email-gate для лид-магнита → платные продукты.
**Проблема:** нет мотивации регистрироваться — всё и так видно.

### Новая модель (v2)
```
Бесплатно (без регистрации):
├── Общая сводка: кол-во нарушений, сумма штрафов, уровень риска
├── Полные данные по ЗАКОНАМ РФ (какие законы нарушены, статьи, штрафы)
├── 2-3 нарушения полностью раскрыты (с описанием и рекомендациями)
├── Остальные нарушения — видны заголовки + штрафы, но детали скрыты (blur)
├── Техническая сводка (SEO/безопасность) — только итоговый балл
└── Маркетинговый блок: "Почему важно исправить + что делать дальше"

После регистрации (бесплатно):
├── ВСЕ нарушения полностью раскрыты с описанием и рекомендациями
├── Полный технический аудит (SEO, безопасность)
├── Проверка сохранена в кабинете, история проверок
├── Ежемесячный мониторинг (бесплатный отчёт на email)
└── Возможность заказать платные услуги

Платные услуги:
├── PDF-отчёт для руководства (подробный, с решениями)
├── "Исправим за вас" (техническое исправление нарушений)
└── "Защита от штрафов" (мониторинг + приоритетная поддержка)
```

## 2. Что показываем БЕЗ регистрации (Freemium-слой)

### Опыт лидеров рынка

| Сервис | Модель | Что работает |
|--------|--------|-------------|
| **GTmetrix** | 5 проверок/мес бесплатно, детали за аккаунт | Лимит создаёт дефицит |
| **SEMrush** | Частичные результаты → "Зарегистрируйтесь чтобы сохранить" | Aha-moment ДО стены |
| **Cookiebot** | 5 страниц бесплатно, все страницы за деньги | Честный лимит > искусственный |
| **Sucuri** | Полный скан бесплатно, исправление платно | Доверие через прозрачность |
| **Pentest-Tools** | Скан бесплатно, мониторинг за аккаунт | Одноразовое ≠ повторяющееся |

### Наша модель: "Закон полностью, техника кратко"

**Логика:** Юридическая информация (какие законы нарушены, какие штрафы грозят) — это публичная информация, скрывать её некрасиво. Но **техническое решение** (что именно исправить в коде, где на сайте проблема) — это экспертиза, за которую разумно просить регистрацию.

#### Показываем без регистрации:
1. **Сводка** — кол-во нарушений, предупреждений, общая сумма штрафов
2. **По законам** — полная разбивка: 152-ФЗ (5 нарушений, до 300К), 38-ФЗ (2 нарушения, до 500К) и т.д.
3. **Первые 3 нарушения** — полностью раскрыты (заголовок + описание + закон + рекомендация)
4. **Остальные нарушения** — видны: заголовок + серьёзность + сумма штрафа. Скрыты: описание, детали, рекомендация (blur-эффект)
5. **Маркетинговый блок** — "У вашего сайта X нарушений на сумму до Y руб. Зарегистрируйтесь бесплатно, чтобы увидеть все детали и получить план исправления"

#### Скрываем (требуют регистрации):
1. Описание и рекомендации для 4+ нарушений
2. Полный технический аудит (SEO: мета-теги, скорость, мобильность)
3. Возможность сохранить отчёт
4. История проверок
5. Повторная проверка без ожидания

#### Логика "экспертные работы — подключение специалиста"

Если при сканировании определяем:
- Хостинг НЕ в РФ (по IP/geo) → предлагаем миграцию (экспертная)
- Язык интерфейса не русский → предлагаем русификацию (экспертная)
- Блог с >1000 подписчиков → предлагаем регистрацию РКН (экспертная)

Для экспертных работ:
- Пометка "Требуется согласование сроков"
- Кнопка "Оставить заявку" вместо "Оплатить"
- Менеджер связывается для уточнения деталей

**После оплаты:**
1. Статус в кабинете: "Ожидаем данные доступа"
2. Форма для ввода SSH/FTP данных (зашифрованные, одноразовые)
3. Работа: автофикс + ручная проверка (для премиум)
4. Повторная проверка → сравнение "до/после"
5. Удаление доступов → подтверждение клиенту

**Для действующих клиентов:** скидка 50% на последующие исправления.

- Отчёт на email + отражение в кабинете
- Если найдены новые нарушения → уведомление + рекомендации
- Возможность заказать исправление со скидкой 50% (действующий клиент)
- Сравнение с предыдущей проверкой (новые/исправленные/повторные)

**Включено в "Исправим за вас":** первые 3 месяца мониторинга бесплатно после покупки пакета исправлений.

**Механика:**
1. Подписка через YooKassa (recurring payment)
2. Cron-задача: раз в месяц → `analyzeUrl()` → сохранить → сравнить → отправить email
3. В кабинете: timeline с историей проверок + тренд-график

## 6. Фичи для повышения конверсии

### 6.1 Compliance Score (0-100)

Визуальный индикатор "здоровья" сайта. Вычисляется по формуле:
- 100 баллов минус штрафные очки за каждое нарушение (critical: -15, high: -10, medium: -5, low: -2)
- Цветовая шкала: 🔴 0-30, 🟠 31-60, 🟡 61-80, 🟢 81-100

**Где показывается:**
- На главной (после проверки): большое число + кольцо прогресса
- В кабинете: на карточке каждого сайта
- В email-отчёте: "Ваш сайт набрал 34 из 100"

**Почему работает:** Числовой балл мотивирует "довести до 100" (геймификация). По данным Appcues, compliance score увеличивает upsell rate на 25%.

### 6.2 Сравнение "до/после"

После исправления (или повторной проверки) — наглядное сравнение:
```
              До (12 апр)        После (15 апр)
Нарушения:    12 🔴              3 🟡
Штрафы:       до 2.5М ₽          до 180К ₽
Score:        34/100              78/100
              ████░░░░░░          ████████░░
```

### 6.3 "Проверьте ваш сайт" виджет для клиентов

Бейдж/виджет, который клиент может разместить на своём сайте:
```
✅ Проверено Штрафометром · Score: 92/100
```
Клик → страница верификации на shtrafometer.ru. Работает как доп. маркетинг.

### 6.4 Уведомления о изменениях законодательства

Для пользователей с мониторингом: "Вступили в силу изменения в 152-ФЗ. Мы проверили ваш сайт — появилось 2 новых требования."

### 6.5 Приоритезация нарушений

В кабинете — не просто список, а Smart-приоритизация:
```
🔴 СРОЧНО (штраф > 100К, проверки идут)
  1. Отсутствие политики ПДн — до 150К ₽
  2. Нет согласия на cookies — до 100К ₽

🟠 ВАЖНО (штраф > 30К)
  3. Нет маркировки рекламы — до 500К ₽
  ...

🟡 РЕКОМЕНДАЦИЯ (штраф < 30К или техническое)
  7. Отсутствует robots.txt
  ...
```

### 6.6 Быстрые действия в один клик

На каждом нарушении в кабинете:
```
[ 📄 В отчёт ]  [ 🔧 Исправить ]  [ ⏭ Пропустить ]
```

## 7. Аналитика механики конверсии

### Воронка v2

```
Посетитель → Проверка сайта (бесплатно, без регистрации)
    ↓ 100%
Видит: сводка + часть нарушений + blur-стена
    ↓ ожидание: 40-60% кликают на скрытое нарушение
Триггер регистрации (модальное окно или CTA-баннер)
    ↓ ожидание: 25-35% регистрируются
Кабинет: полные результаты + история + рекомендации
    ↓ ожидание: 15-25% заказывают PDF
    ↓ ожидание: 5-10% заказывают исправление
    ↓ ожидание: 10-15% подключают мониторинг
```

### Метрики для отслеживания
- **Check-to-register rate** — % прошедших от проверки к регистрации (цель: >30%)
- **Register-to-pay rate** — % оплативших хотя бы одну услугу (цель: >15%)
- **Monitoring retention** — % продливших подписку на 2+ месяц (цель: >60%)
- **Repeat purchase rate** — % заказавших вторую услугу (цель: >20%)

```

```

### 8.4 Модель данных

**Новая таблица `check_history`:**
```sql
CREATE TABLE check_history (
  id TEXT PRIMARY KEY,
  user_id TEXT REFERENCES users(id),
  domain TEXT NOT NULL,
  checked_at TIMESTAMPTZ DEFAULT NOW(),
  violations INTEGER,
  warnings INTEGER,
  passed INTEGER,
  total_max_fine INTEGER,
  compliance_score INTEGER,      -- 0-100
  check_result JSONB,
  -- diff с предыдущей проверкой
  new_violations INTEGER DEFAULT 0,
  fixed_violations INTEGER DEFAULT 0,
  recurring_violations INTEGER DEFAULT 0
);
```
