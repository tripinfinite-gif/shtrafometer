## Архитектура кабинета v2

### Изменения относительно v1.5 (и почему)

**Было:** Простой sidebar с 4 пунктами (Главная, Сайты, Заказы, Настройки).

**Стало:** Site-centric архитектура (как Cloudflare, Google Search Console).

| Изменение | Было | Стало | Почему |
|-----------|------|-------|--------|
| **Навигация** | Sidebar | Dock сверху + sidebar на странице сайта | Cloudflare паттерн: работает для 1 и для 100 сайтов |
| **Главная кабинета** | Дашборд со списком | Site-selector + overview | GSC паттерн: сразу видишь здоровье каждого сайта |
| **Проверка** | Отдельная страница | Вкладка внутри сайта | Не нужно выходить из контекста (Ahrefs паттерн) |
| **Empty state** | "У вас нет сайтов" | "Проверьте первый сайт за 2 минуты" + иллюстрация | Sucuri паттерн: пустое состояние = onboarding |
| **Услуги** | Общий список | Привязаны к конкретному сайту | Логика: исправляем КОНКРЕТНЫЙ сайт, а не абстрактно |
| **Upsell** | Отдельные продукт-карточки | Контекстные CTA рядом с каждым нарушением | Semrush паттерн: предлагай решение там, где показана проблема |

### Compliance Score (новая фича)

Числовой балл 0-100, вычисляется автоматически:
```
100 - (critical × 15) - (high × 10) - (medium × 5) - (low × 2)
Минимум: 0
```

Цветовая шкала: 0-30 красный, 31-60 оранжевый, 61-80 жёлтый, 81-100 зелёный.

**Где показывается:**
- На главной (после проверки) — большое кольцо
- В кабинете — на карточке каждого сайта
- В email-отчёте мониторинга

**Почему:** Числовой балл мотивирует "довести до 100" (геймификация). По данным Appcues, +25% к upsell.

### Тегирование нарушений (новое / исправлено / повторное)

При каждой проверке сравниваем с предыдущей:
- **Новое** — появилось впервые
- **Исправлено** — было, теперь нет
- **Повторное** — было и осталось

**Почему:** Detectify/Acunetix паттерн. Клиент видит прогресс, мотивирует продолжать.

## Freemium-гейтинг на главной

### Что показываем без регистрации
1. **Compliance Score** — большое кольцо с баллом
2. **Сводка** — кол-во нарушений, предупреждений, сумма штрафов
3. **По законам** — полная разбивка: какой закон, сколько нарушений, штрафы
4. **Первые 3 нарушения** — полностью (заголовок + описание + рекомендация)
5. **Остальные нарушения** — видно: заголовок + серьёзность + штраф. Blur на деталях.
6. **Техническая сводка** — только итоговый балл, без деталей

### Что скрываем (требуют регистрации)
1. Описание и рекомендации для 4+ нарушений
2. Полный технический аудит
3. Сохранение отчёта
4. История проверок

### Триггеры регистрации (от мягкого к настойчивому)

1. **Blur-стена** на 4+ нарушении — кнопка "Зарегистрироваться бесплатно"
2. **CTA-баннер** после списка нарушений — "Получите полный отчёт бесплатно"
3. **Sticky-бар** на мобильном — "12 нарушений · [ Полный отчёт → ]"
4. **Модалка** при клике на скрытое нарушение — форма регистрации

### Антипаттерны (НЕ делаем)
- Не блокируем ВСЕ результаты (ошибка GTmetrix — потеряли пользователей)
- Не прячем юридическую информацию (законы — публичные)
- Не требуем email на первом шаге (только телефон)
- Не показываем пустую страницу за логином

## Фичи для конверсии

### Приоритезация нарушений (Smart Priority)

Не просто список, а сортировка по срочности:
```
🔴 СРОЧНО (штраф > 100К, проверки идут)
🟠 ВАЖНО (штраф > 30К)
🟡 РЕКОМЕНДАЦИЯ (штраф < 30К или техническое)
```

### Контекстные CTA на каждом нарушении (Semrush паттерн)

Рядом с каждым нарушением в кабинете:
```
[ 📄 В отчёт ]  [ 🔧 Исправить ]  [ ⏭ Пропустить ]
```

**Почему:** Semrush показал, что контекстный upsell (предложение решения рядом с проблемой) конвертирует в 3x лучше, чем отдельная страница продуктов.

### Сравнение "до/после"

После исправления или повторной проверки:
```
              До (12 апр)        После (15 апр)
Нарушения:    12                 3
Штрафы:       до 2.5М руб       до 180К руб
Score:        34/100             78/100
```

### Credential Trust Flow (Cloudways/Sucuri паттерн)

Прогрессивное доверие — не просим доступ сразу:
1. **Шаг 1:** Добавьте URL (никаких паролей)
2. **Шаг 2:** Проверка запущена (только чтение, public HTML)
3. **Шаг 3:** Результаты готовы (доверие построено)
4. **Шаг 4:** Хотите исправить? → тогда нужен доступ (SSH/FTP)
5. **Шаг 5:** Доступ предоставлен → исправляем → доступ удаляем
