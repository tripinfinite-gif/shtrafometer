# Инструкция для агента: регистрация OAuth-приложения Яндекс.Директ

> Для Comet / Claude-in-Chrome или любого браузерного агента с доступом к аккаунту Яндекса владельца Штрафометра (tripinfinite@gmail.com или корп).
> Цель: получить `client_id`, `client_secret` и подать заявку в реестр API Директа.

---

## Контекст (прочитай перед началом)

- Проект: **Штрафометр** (shtrafometer.ru), SaaS проверки сайтов на соответствие законам РФ.
- Нужно: встроить в админку (`/admin/ai-consultant`) AI-консультанта, который сможет читать реальные данные из Яндекс.Директа, Метрики и Вебмастера **через API**.
- Это требует OAuth-приложения в кабинете Яндекса + отдельной заявки в реестр API Директа (отдельная модерация 1-3 рабочих дня).
- Все mutating-действия в приложении идут через HITL-подтверждение, т.е. приложение безопасно для бюджета.

---

## Что нужно сделать (по шагам)

### Шаг 1. Регистрация OAuth-приложения на oauth.yandex.ru

1. Открой **https://oauth.yandex.ru/** (логин — аккаунт владельца Штрафометра; уточни у пользователя какой использовать).
2. Нажми **«Зарегистрировать новое приложение»**.
3. Заполни форму:
   - **Название сервиса:** `Shtrafometer AI Consultant`
   - **Иконка:** пропусти (можно позже)
   - **Платформы:** выбери **«Веб-сервисы»**
     - **Callback URI #1:** `https://shtrafometer.ru/api/admin/ai/oauth/yandex/callback`
     - Нажми «Добавить ещё» → **Callback URI #2:** `http://localhost:3000/api/admin/ai/oauth/yandex/callback` (для dev)
   - **Доступ к данным (scopes):** отметь ВСЕ из списка ниже:
     - ✅ `direct:api` — Использование API Яндекс.Директа
     - ✅ `metrika:read` — Получение статистики Метрики
     - ✅ `metrika:write` — (если будет в списке; если нет — пропусти)
     - ✅ `webmaster:hostinfo` — Управление сайтами в Вебмастере
     - ✅ `webmaster:verify` — Подтверждение прав на сайт
4. **«Электронная почта для связи»:** `tripinfinite@gmail.com` (или корп-email, спроси пользователя).
5. Нажми **«Сохранить»**.
6. На странице созданного приложения скопируй:
   - **ClientID** (длинная строка)
   - **Client secret** (нажми «Показать» — длинная строка)
7. **Отдай их пользователю в чате** открытым текстом в формате:
   ```
   YANDEX_DIRECT_CLIENT_ID=<...>
   YANDEX_DIRECT_CLIENT_SECRET=<...>
   YANDEX_DIRECT_REDIRECT_URI=https://shtrafometer.ru/api/admin/ai/oauth/yandex/callback
   ```

---

### Шаг 2. Подача заявки в реестр API Яндекс.Директа

ВАЖНО: **OAuth-приложение само по себе НЕ даёт доступа к API Директа.** Нужна отдельная модерация заявки.

1. Зайди в **https://direct.yandex.ru/** (тем же аккаунтом владельца).
2. Если рекламного кабинета нет — создай его (пустой, можно не запускать кампании).
3. В левом меню или в шапке найди **«Инструменты» → «API»** (или: Настройки → API → «Получить доступ»).
4. Заполни заявку:
   - **Название приложения:** `Shtrafometer AI Consultant`
   - **ClientID приложения:** вставь ClientID из Шага 1
   - **Тип доступа:** **Full** (полный — и чтение, и изменение). Если предложат «Только чтение» — НЕ выбирай, нам нужен full для managed actions с HITL.
   - **URL сервиса:** `https://shtrafometer.ru`
   - **Описание:** скопируй следующий текст дословно:
     ```
     AI-консультант во внутренней админпанели проекта Шtrafometer (SaaS проверки сайтов
     на соответствие законам РФ). Консультант помогает администратору анализировать
     рекламные кампании Яндекс.Директ, читать статистику, предлагать оптимизации
     креативов и ключевых фраз. Все изменяющие действия выполняются только
     после подтверждения оператором (human-in-the-loop). Приложение предназначено
     для внутреннего использования владельцем сервиса, не публикуется для третьих лиц.
     ```
   - **Тип клиента:** «Агентство» НЕТ, выбери **«Рекламодатель»**.
5. Отправь заявку.
6. **Зафиксируй дату подачи** и сообщи пользователю: «Заявка подана <дата>, модерация 1-3 рабочих дня, подтверждение придёт на email <email>».

---

### Шаг 3. (опционально, параллельно) Подключение Метрики и Вебмастера

Обычно scopes из Шага 1 достаточно, отдельной заявки для Метрики/Вебмастера не нужно. Проверь:
- Счётчик Метрики **108525306** (Штрафометр) привязан к тому же аккаунту — если нет, привяжи.
- Сайт `shtrafometer.ru` добавлен в Вебмастере того же аккаунта — если нет, добавь и подтверди права (метатег или HTML-файл; файл `yandex_6ee371817dc2e30f.html` уже в `/public/`).

---

## Важные правила

- **НЕ создавай рекламные кампании, НЕ трать бюджет.**
- **НЕ меняй пароли и настройки аккаунта.**
- Если Яндекс попросит подтверждение по SMS — **остановись и спроси пользователя**, не подтверждай сам.
- Если форма API-заявки требует поля, которых нет в этой инструкции — **спроси пользователя**, не придумывай.
- Если `oauth.yandex.ru` покажет предупреждение «у вас уже есть приложение с таким названием» — **не создавай дубликат**, сообщи пользователю и спроси, какое использовать.
- Скриншоты каждого ключевого шага (создание приложения, заявка API) — приложи к отчёту.
