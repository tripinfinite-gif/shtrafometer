# Настройка Яндекс 360 для email-рассылок

## Промпт для Claude в Chrome

```
Помоги мне подключить почту на домене shtrafometer.ru через Яндекс 360 для бизнеса.

Мне нужно создать почтовый ящик noreply@shtrafometer.ru для отправки писем из приложения через SMTP.

Пошагово:

1. Зайти на admin.yandex.ru (или business.yandex.ru/mail360)
2. Если нет организации — создать (название: Штрафометр)
3. Добавить домен shtrafometer.ru:
   - Домены → Добавить домен → shtrafometer.ru
4. Подтвердить владение доменом — Яндекс предложит один из способов:
   - РЕКОМЕНДУЮ: TXT-запись в DNS
   - Зайти на reg.ru → DNS → добавить TXT-запись как указано Яндексом
5. Настроить MX-запись на reg.ru:
   - Тип: MX
   - Хост: @
   - Значение: mx.yandex.net.
   - Приоритет: 10
6. Добавить SPF-запись (если нет):
   - Тип: TXT
   - Хост: @
   - Значение: v=spf1 redirect=_spf.yandex.net
7. Добавить DKIM-запись (Яндекс покажет значение):
   - Тип: TXT
   - Хост: mail._domainkey
   - Значение: (скопировать из Яндекса)
8. Дождаться подтверждения домена (5-30 минут)
9. Создать почтовый ящик:
   - Сотрудники → Добавить сотрудника
   - Логин: noreply
   - Имя: Штрафометр
   - Фамилия: Рассылки
   - Установить пароль (ЗАПИШИ ЕГО!)
10. Также создать:
   - info@shtrafometer.ru (для входящей почты, контактов)

ВАЖНО:
- Для SMTP из приложения нужен "пароль приложения":
  - Зайти в аккаунт noreply@shtrafometer.ru на mail.yandex.ru
  - Настройки → Безопасность → Пароли приложений → Создать пароль для "Почта"
  - Этот пароль использовать как SMTP_PASS

После создания скинь мне:
- Пароль приложения для noreply@shtrafometer.ru

Я добавлю его на сервер в переменные:
- SMTP_HOST=smtp.yandex.ru
- SMTP_PORT=465
- SMTP_USER=noreply@shtrafometer.ru
- SMTP_PASS=[пароль приложения]

Веди меня пошагово. Если скину скриншот — подскажи что нажимать.
```

## Какие DNS-записи добавить на reg.ru

| Тип | Хост | Значение | Назначение |
|-----|------|----------|------------|
| MX | @ | mx.yandex.net. (приоритет 10) | Приём почты |
| TXT | @ | v=spf1 redirect=_spf.yandex.net | SPF (защита от спуфинга) |
| TXT | mail._domainkey | (из Яндекса) | DKIM (подпись писем) |
| TXT | @ | (верификация домена — из Яндекса) | Подтверждение владения |

Существующие A-записи (@ → 109.69.18.80, www → 109.69.18.80) — НЕ ТРОГАТЬ.

## После настройки

Скинь мне пароль приложения — я добавлю в docker-compose на сервере и передеплою.
